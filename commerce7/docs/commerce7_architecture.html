Sync Strategy

Scheduled Sync — every 15 minutes
The scheduler fires run_sync(). All enabled resource types are fetched in full from Commerce7 using cursor-based pagination (no rate limit). Commerce7 lacks a native delta/changelog endpoint, so a full re-fetch is always done.
Smart Deduplication via updatedAt
Each record's updatedAt timestamp is stored as the external_revision_id. DataSourceEntitiesProcessor compares this on upsert — if unchanged, the record is skipped, saving compute and avoiding unnecessary re-embedding.
Webhook Support (real-time)
Commerce7 webhooks can trigger handle_webhook_notification() on Customer, Order, or Product create/update/delete events — enabling near-real-time sync without waiting for the next 15-minute window.
SyncPoint Tracking
After each resource type syncs, a SyncPoint is saved in ArangoDB with a timestamp and record count — giving you a full audit trail of every sync run.
Permissions
All synced records get an org-wide READER permission — every user in your PipesHub AI org can search and query Commerce7 data. No per-user permission mapping needed since Commerce7 is B2B SaaS.
